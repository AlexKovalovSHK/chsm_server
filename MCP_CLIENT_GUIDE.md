# Руководство по интеграции с MCP сервером (CHSM Classroom)

Данный документ описывает принцип работы и способы взаимодействия с Model Context Protocol (MCP) сервером, интегрированным в приложение CHSM Classroom.

---

## 1. Общий принцип работы

Сервер реализует **Model Context Protocol (MCP)** — открытый стандарт, позволяющий AI-моделям (Claude, GPT, Gemini) безопасно взаимодействовать с локальными и удаленными данными и инструментами.

### Архитектура:
1.  **Инструменты (Tools)**: Выполняемые функции (например, "получить список студентов").
2.  **Транспорты**: 
    *   **SSE (Server-Sent Events)**: Основной транспорт для веб-клиентов и внешних интеграций.
    *   **STDIO**: Для локального использования AI-агентами (например, Claude Desktop).
3.  **Безопасность**: Все запросы (кроме STDIO) защищены JWT-авторизацией и проверяются через `AdminAndTeacherGuard`.

---

## 2. Получение конфигурации (Discovery)

Для получения списка доступных инструментов и моделей без установки полноценного MCP-соединения используйте упрощенный эндпоинт.

**Запрос:**
`GET /chat/config`

**Заголовки:**
`Authorization: Bearer <ваш_jwt_токен>`

**Ответ:**
```json
{
  "appName": "CHSM-Classroom-MCP",
  "models": ["gemini", "deepseek"],
  "tools": [
    {
      "name": "list-students",
      "description": "Получить список всех студентов."
    },
    {
      "name": "get-student",
      "description": "Получить детальную информацию о студенте по ID."
    }
  ]
}
```

## 3. Взаимодействие через SSE (Протокол MCP)

Для полноценного взаимодействия клиент должен следовать протоколу MCP.

### Шаг 1: Установка соединения
Клиент открывает SSE-поток. Сервер выдает уникальный ID сессии.

**URL:** `GET /mcp/sse`
**Заголовок:** `Authorization: Bearer <token>`

При подключении сервер отправит событие `endpoint`:
```
event: endpoint
data: /mcp/messages?sessionId=xxxx-xxxx-xxxx
```

### Шаг 2: Отправка команд (JSON-RPC)
Все последующие действия (получение списка инструментов, вызов функции) выполняются через `POST` на URL, полученный в событии `endpoint`.

**Пример вызова инструмента:**
`POST /mcp/messages?sessionId=xxxx-xxxx-xxxx`

```json
{
  "jsonrpc": "2.0",
  "method": "tools/call",
  "params": {
    "name": "get-student",
    "arguments": {
      "studentId": "3b2e5... (UUID)"
    }
  },
  "id": 1
}
```

## 4. Прямой AI Чат (AI Bridge)

Вы можете взаимодействовать с MCP инструментами через встроенный AI-чат. В этом случае AI сам решит, какой инструмент вызвать для выполнения вашего запроса.

**URL:** `POST /chat`
**Заголовки:** `Authorization: Bearer <token>`

**Тело запроса:**
```json
{
  "model": "gemini",
  "messages": [
    {
      "role": "user",
      "content": "Найди студента с ID 123 и скажи как его зовут"
    }
  ]
}
```

**Логика выполнения:**
1. Сервер получает запрос.
2. AI анализирует запрос и видит доступный инструмент `get-student`.
3. AI вызывает инструмент внутри сервера.
4. Вы получаете итоговый ответ на человеческом языке.

---

## 5. Доступные инструменты (Tools)

### `list-students`
*   **Описание**: Возвращает массив всех студентов в системе.
*   **Параметры**: Нет.

### `get-student`
*   **Описание**: Возвращает полную карточку студента.
*   **Параметры**:
    *   `studentId` (string, required): UUID студента.

---

## 6. Требования к авторизации

Все запросы должны содержать заголовок:
`Authorization: Bearer <JWT>`

JWT должен содержать в payload роль:
*   `admin`
*   `teacher`

При отсутствии токена или недостаточности прав сервер вернет `401 Unauthorized` или `403 Forbidden`.
